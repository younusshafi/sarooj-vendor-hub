Procurement AI — Progress Report

Period: 22–23 June 2026 Prepared by: Engineering Audience: Procurement team
Executive summary. In this cycle we (1) merged the separate Subcontractor RFQ tool into the main procurement app so everything lives in one place, (2) made several day-to-day fixes to the RFQ and bid-comparison screens, and (3) built and connected a new end-to-end bidding and award workflow that lets vendors submit priced quotations through a secure web link, lets the procurement officer equalize and award line by line, and routes the result to the Procurement Head (Rabia) for final approval, after which it locks. The new workflow is built and connected to the live database. The only remaining piece is the email automation (sending vendors their link and emailing Rabia for approval), which sits with the backend/automation operator. Until that is switched on, the links work but must be shared manually.

1. One unified application (Materials + Subcontractor)

The standalone Subcontractor RFQ app has been folded into the main app. You now use a single system for both Materials (MR) and Subcontractor (SR) RFQs:

  • One sidebar, one dashboard, one RFQ tracker listing both types.
  • Subcontractor RFQs keep their own create and detail screens within the same app.
  • No change to the existing Materials flows — they work exactly as before.

A small clean-up: on the Subcontractor RFQ screen, the “Frame” tab is renamed “BoQ Upload”, and the action button is now “Lock & Build BoQ” (clearer language for the same step). Once an SR RFQ is issued, its Overview details and covering email become read-only to prevent accidental edits after dispatch.

Note: the old standalone Subcontractor app can be decommissioned once you’ve confirmed everything works in the unified app.

2. RFQ & vendor screen improvements

  • Vendor tab now shows the real recipients. Previously, after an RFQ was issued the Vendors tab listed the entire matched vendor pool. It now shows only the vendors actually sent the RFQ (“Sent to N vendors”), with a “Show all matched” toggle for the wider pool. This matches what actually went out and removes confusion. (Drafts still show the full selectable list as before.)

3. Bid comparison improvement

  • Lowest bid is now highlighted in green instead of the previous pink/crimson. Green reads correctly as “best price”; the old colour looked like a warning.

4. New: deterministic bidding, line-by-line award, and approval

This is the major addition. It replaces error-prone AI reading of emailed quotes with structured, vendor-entered data, and adds the award and sign-off steps procurement asked for.

a) Vendors submit quotations via a secure link

  • Each invited vendor receives their own single-use web link. They open it (no login), see the material list, and type their unit rates, brand/make, quantity offered, and a remark/exclusion per line, plus commercial terms (payment, lead time, validity, VAT).
  • They can leave lines blank (“no quote”), and revise and re-submit until the deadline; it then locks.
  • On submit, the data lands directly in the system and flows straight into the comparison — no manual entry, no AI guesswork.
  • The link is the only way to quote, which keeps the data clean and comparable.

b) Officer equalization (fair, like-for-like comparison)

  • When a vendor’s remark says they exclude part of the scope (e.g. “we won’t do X”), the officer can add an “equalization” value — a budgeted cost for the missing scope — so that vendor’s line is compared fairly against vendors who included it.
  • The vendor’s original quote is never changed; the equalization is the officer’s judgement, with a required note. Rankings (and the green “lowest”) use the equalized figure.

c) Line-by-line award

  • The officer awards each line to a vendor, defaulting to the lowest equalized price. A 20-line RFQ can be split across several vendors (e.g. 5 lines each to three vendors).
  • If a line is not awarded to the lowest, a reason is required.
  • A split summary shows each vendor’s awarded lines and totals — the officer’s worksheet for raising POs.

d) Two-step approval (Officer → Procurement Head)

  • When ready, the officer clicks “Submit for approval.” This generates a single-use review link for Rabia.
  • Rabia opens the link, sees the proposed award (per-line, equalized, with reasons and totals), and either Approves & locks or Returns for revision with comments.
  • On approval, the comparison and awards are locked at the database level — no further edits. If returned, the officer sees the comments and can revise and resubmit.

Scope note: raising the actual Purchase Orders remains a manual step by the officer for now (PO generation was intentionally left out of this phase). The Subcontractor (SR) version of this bidding/award/approval flow is planned next and needs a couple of SR-specific decisions before it’s built.

5. Data sanity check (21 June upload — “7 PRs but 5 showed”)

We investigated the report that a material-sheet upload said ~7 PRs but the dashboard appeared to show fewer. Result: no data was lost. All 7 PRs from the 21 June upload are correctly stored and visible on the PR Tracker. The “fewer” appearance came from the dashboard’s “Recent RFQs (latest 5)” widget, which deliberately shows only the 5 newest. Two reminders: the dashboard “PRs tracked” figure is a running total across all dates, and one RFQ can span multiple PRs (and vice-versa), so RFQ count and PR count are not the same number. (Optional future tweak: add a “View all” link / “showing 5 of N” label.)

7. How to try it (once deployed/verified)

  • Open an issued Materials RFQ with bids → Compare bids.
  • Use Per-line award & equalization: adjust any excluded-scope line, award each line, Save.
  • Submit for approval → copy the review link → open it as “Rabia” → Approve & lock or Return.
  • Vendor experience: open a bid link → fill the material list → Submit.

8. Notes for awareness

  • The new features are connected to the live database; the underlying data tables and rules were added without affecting any existing data or flows.
  • A couple of items are flagged for the backend/IT operator: the new database objects now exist in the live system, and the database currently runs with broad access settings — worth a security review as the platform grows.
Bottom line. Materials procurement now has a complete digital path from vendor quotation → fair comparison → line-by-line award → Procurement Head approval → locked record, with deterministic data instead of AI extraction. It is built and database-connected; it goes fully self-serve for vendors the moment the email automation is switched on. The Subcontractor version is the next step.
